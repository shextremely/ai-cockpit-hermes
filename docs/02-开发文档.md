## 1. 仓库结构

```text
e:/ai-cockpit-hermes/
├─ package.json              # workspaces 根脚本
├─ .env / .env.example       # 环境变量（根目录，BFF 自动加载）
├─ docs/                     # 设计/开发/使用文档
├─ apps/
│  ├─ bff/                   # Node + Express + TypeScript
│  │  └─ src/
│  │     ├─ index.ts         # 入口、路由挂载、静态托管
│  │     ├─ config.ts        # 环境变量
│  │     ├─ hermes.ts        # Hermes HTTP 客户端
│  │     ├─ sse.ts           # SSE 透传
│  │     ├─ middleware/auth.ts
│  │     └─ routes/          # health, runs, dashboard, jobs, knowledge, capabilities
│  └─ web/                   # Vue 3 + Vite + TypeScript
│     └─ src/
│        ├─ main.ts / App.vue
│        ├─ router/          # 五页面路由
│        ├─ stores/          # chat, capabilities
│        ├─ api/             # client, sse
│        ├─ layouts/AppLayout.vue
│        ├─ components/      # ChatDrawer, HermesStatus, ...
│        └─ views/           # Dashboard, Tfs, Knowledge, Jobs, Capabilities
```

## 2. 技术选型

| 层级 | 技术 | 说明 |
| --- | --- | --- |
| 前端 | Vue 3 `<script setup>` + TS | Composition API |
| 构建 | Vite 6 | HMR；开发端口 **5174** |
| 状态 | Pinia | chat / capabilities |
| UI | Naive UI | 暗色主题 |
| 路由 | Vue Router | 懒加载页面 |
| SSE | `fetch` + `ReadableStream` | 支持 POST + 自定义头（非 EventSource） |
| Markdown | markdown-it | 渲染 Agent 回复 |
| BFF | Express 4 + tsx | 开发热重载 |
| 通信 | 原生 `fetch` | 调 Hermes OpenAI 兼容 API |

## 3. 环境变量

根目录 `.env`（参考 `.env.example`）：

```bash
# Hermes（与 ~/.hermes/.env 中 API_SERVER_* 对应）
HERMES_BASE_URL=http://127.0.0.1:5173
HERMES_API_KEY=<与 API_SERVER_KEY 相同>
HERMES_SESSION_KEY=agent:main:cockpit:user-self
HERMES_MODEL=hermes-agent

# BFF
BFF_PORT=5180
BFF_ALLOW_ORIGIN=http://localhost:5174
BFF_AUTH_TOKEN=                    # 可选，启用后前端需带 X-Cockpit-Token

# 前端（禁止放密钥）
VITE_BFF_BASE_URL=/api
```

**红线**：`HERMES_API_KEY` 不得使用 `VITE_` 前缀，不得出现在 `apps/web` 源码中。

`apps/bff/src/config.ts` 加载顺序：仓库根 `.env` → `apps/bff/.env`（后者覆盖）。

## 4. BFF API 契约

所有接口前缀 `/api`，开发期由 Vite 代理到 `localhost:5180`。

### 4.1 健康与能力

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/health` | BFF + Hermes 连通性 |
| GET | `/api/health/detailed` | Hermes 详细状态 |
| GET | `/api/capabilities` | 透传 `/v1/capabilities` |
| GET | `/api/skills` | 透传 `/v1/skills` |
| GET | `/api/toolsets` | 透传 `/v1/toolsets` |

### 4.2 对话（Runs 主链路）

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/chat` | Body: `{ input, sessionId?, instructions? }` → 创建 run |
| GET | `/api/runs/:id/events` | SSE 透传 Hermes 运行事件 |
| GET | `/api/runs/:id` | 轮询运行状态 |
| POST | `/api/runs/:id/stop` | 中断运行 |
| POST | `/api/runs/:id/approval` | 审批门禁 Body 透传 |

### 4.3 业务聚合

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/dashboard` | 首页 TFS 卡片 JSON（经 Hermes chat/completions） |
| POST | `/api/knowledge/search` | Body: `{ query }` → Obsidian 检索结果 |

### 4.4 定时任务（透传 Hermes Jobs）

| 方法 | 路径 |
| --- | --- |
| GET/POST | `/api/jobs` |
| GET/PATCH/DELETE | `/api/jobs/:id` |
| POST | `/api/jobs/:id/pause` · `/resume` · `/run` |

## 5. Hermes 集成

### 5.1 密钥来源

`API_SERVER_KEY` **不是**向服务商申请，而是在 `~/.hermes/.env` **自行设定**：

```bash
API_SERVER_ENABLED=true
API_SERVER_KEY=你的密钥
API_SERVER_PORT=5173
API_SERVER_HOST=127.0.0.1
```

驾驶舱 `.env` 中 `HERMES_API_KEY` 必须与之一致。启动：`hermes gateway`。

### 5.2 端点映射（BFF → Hermes）

| BFF | Hermes |
| --- | --- |
| POST `/api/chat` | POST `/v1/runs` |
| GET `/api/runs/:id/events` | GET `/v1/runs/:id/events` |
| GET `/api/dashboard` | POST `/v1/chat/completions` |
| `/api/jobs/*` | `/api/jobs/*` |
| `/api/capabilities` 等 | `/v1/capabilities` 等 |

### 5.3 请求头

BFF 在调 Hermes 时自动附加：

```http
Authorization: Bearer <HERMES_API_KEY>
X-Hermes-Session-Key: <HERMES_SESSION_KEY>
X-Hermes-Session-Id: <可选，按会话传入>
```

### 5.4 MCP 接入

在 Hermes 配置中注册 MCP Server（TFS、Obsidian、DevOps 等），驾驶舱不直连 MCP。验证：

```bash
curl -H "Authorization: Bearer $KEY" http://127.0.0.1:5173/v1/toolsets
```

### 5.5 SSE 透传要点

`apps/bff/src/sse.ts`：

- `Content-Type: text/event-stream`
- `Cache-Control: no-cache`
- `X-Accel-Buffering: no`
- 逐块 `res.write`，禁止响应缓冲

前端 `apps/web/src/api/sse.ts` 用 `fetch` 解析 `\n\n` 分隔的 SSE 帧。

### 5.6 能力降级

启动时 `useCapabilitiesStore.load()` 读 `/api/capabilities`：

- 无 `run_events_sse` → 可降级 `chat/completions` stream
- 无 Jobs → 隐藏日程模块相关操作

## 7. 开发与构建命令

```bash
# 安装（仓库根目录）
npm install

# 开发：BFF(5180) + 前端(5174) 并行
npm run dev

# 类型检查
npm run typecheck

# 生产构建
npm run build

# 生产启动（BFF 托管 web/dist，访问 :5180）
npm start
```

### Vite 代理

`apps/web/vite.config.ts` 将 `/api` 代理到 `BFF_PORT`（默认 5180）。前端开发访问 **http://localhost:5174**。

### 端口约定

| 服务 | 端口 |
| --- | --- |
| Hermes | 5173（用户自定义，勿与 Vite 冲突） |
| Vite | 5174（`strictPort: true`） |
| BFF | 5180 |

## 8. 编码规范

- TypeScript `strict`；提交前 `npm run typecheck`
- 组件统一 `<script setup lang="ts">`
- HTTP 调用集中在 `api/` 与 BFF `routes/`，不散落组件内
- 注释解释「为什么」，不复述代码
- `.env` 不入 Git（已在 `.gitignore`）

## 9. 扩展指南

### 新增业务页面

1. `apps/web/src/views/XxxView.vue`
2. `router/index.ts` 注册路由
3. `AppLayout` 侧栏增加菜单项
4. 如需新 API：`apps/bff/src/routes/xxx.ts` + `index.ts` 挂载

### 新增 Hermes 结构化卡片

1. 在 BFF 增加 route，system 提示约束 JSON 输出
2. 前端解析 JSON 渲染 `DashboardCard`
3. 解析失败时回退 `raw` 文本展示

### 对接新 MCP

仅在 Hermes 侧配置 MCP；驾驶舱通过自然语言或结构化 prompt 引导 Agent 调用，无需改 BFF（除非要做专用 REST 聚合）。

## 10. 已知限制

- `dashboard` / `knowledge/search` 使用 `chat/completions` 非流式，依赖模型返回合法 JSON
- approval 请求体字段以实际 Hermes 版本为准，需联调对齐 `stores/chat.ts`
- Windows 访问 WSL2 内 Hermes 依赖端口转发；不通时可将 BFF 也部署在 WSL2
