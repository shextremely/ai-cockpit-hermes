# 个人 AI 驾驶舱 — 使用文档

> 面向使用者的安装、配置、启动与日常操作指南。

| 项 | 内容 |
| --- | --- |
| 文档版本 | v1.0 |
| 项目路径 | `e:/ai-cockpit-hermes` |
| 适用环境 | Windows + WSL2（Hermes）+ Node.js 20+ |

---

## 1. 系统组成

使用前需理解三个进程：

| 组件 | 作用 | 你怎么访问 |
| --- | --- | --- |
| **Hermes Agent** | AI 引擎 + 工具编排 | 不直接访问，由 BFF 代理 |
| **驾驶舱 BFF** | 安全代理、持有密钥 | 开发期 `localhost:5180/api` |
| **驾驶舱前端** | 网页界面 | 开发期 **http://localhost:5174** |

生产模式合并为一步：构建后只访问 **http://localhost:5180**。

## 2. 前置条件

### 2.1 已安装

- [ ] Node.js 20+（`node -v`）
- [ ] WSL2 + Hermes Agent（`hermes gateway` 可运行）
- [ ] Hermes 已配置 LLM 提供商（Nous Portal / OpenRouter 等）
- [ ] （可选）TFS / Obsidian MCP 已接入 Hermes

### 2.2 端口规划（当前推荐）

| 服务 | 端口 | 配置文件 |
| --- | --- | --- |
| Hermes API | **5173** | `~/.hermes/.env` → `API_SERVER_PORT` |
| 驾驶舱前端 | **5174** | `apps/web/vite.config.ts` |
| 驾驶舱 BFF | **5180** | `.env` → `BFF_PORT` |

> **注意**：Hermes 与 Vite 不能共用同一端口。若你把 Hermes 改成 5173，前端会自动使用 5174。

## 3. 第一步：配置 Hermes

> Windows 原生安装见 **[04-Hermes启动与自启.md](04-Hermes启动与自启.md)**（含登录自启、本机路径说明）。  
> 以下为 WSL2 环境参考。

在 **WSL2** 中编辑 `~/.hermes/.env`：

```bash
API_SERVER_ENABLED=true
API_SERVER_KEY=shextremely          # 自行设定，下文驾驶舱要填相同值
API_SERVER_PORT=5173
API_SERVER_HOST=127.0.0.1
```

启动网关：

```bash
hermes gateway
```

成功时应看到：

```text
[API Server] API server listening on http://127.0.0.1:5173
```

### 验证 Hermes

在 WSL2 内：

```bash
curl -s http://127.0.0.1:5173/health
curl -s -H "Authorization: Bearer shextremely" http://127.0.0.1:5173/v1/models
```

在 Windows PowerShell（WSL 端口转发正常时）：

```powershell
curl http://127.0.0.1:5173/health
```

### 密钥说明

- `API_SERVER_KEY` **不是**网上申请的，是你在 Hermes 里**自己设定**的本地 API 口令。
- 与 Nous Portal / OpenRouter 的模型 Key **不是同一个东西**。
- 生成强随机串（可选）：`openssl rand -hex 32`

## 4. 第二步：配置驾驶舱

在 `e:/ai-cockpit-hermes`：

```bash
copy .env.example .env
```

编辑 `.env`（关键项）：

```bash
HERMES_BASE_URL=http://127.0.0.1:5173
HERMES_API_KEY=shextremely              # 必须与 API_SERVER_KEY 一致
BFF_PORT=5180
BFF_ALLOW_ORIGIN=http://localhost:5174
```

## 5. 第三步：安装与启动

```bash
cd e:/ai-cockpit-hermes
npm install
npm run dev
```

启动成功后：

- 浏览器打开：**http://localhost:5174/**
- 顶栏 **Hermes 在线** 应为绿色（若红色，见 [第 8 节排障](#8-常见问题)）

### 生产模式（可选）

```bash
npm run build
npm start
# 访问 http://localhost:5180
```

## 6. 日常使用

### 6.1 界面导航

| 菜单 | 功能 |
| --- | --- |
| 驾驶舱 | TFS 概览、任务列表 |
| TFS 工作台 | 三列看板（需求/进行中/BUG） |
| 知识库 | 输入关键词检索 Obsidian Vault |
| 日程待办 | 管理 Hermes 定时 Jobs |
| 能力面板 | 查看 Hermes 技能、工具集、特性开关 |

### 6.2 对话抽屉

- 点击顶栏 **「对话」** 或按 **`Ctrl/Cmd + K`**
- 输入自然语言，例如：
  - 「我今天有哪些 TFS 任务？」
  - 「这周有哪些超期需求？」
  - 「在知识库里搜一下 TFS 周报怎么写」
- 发送后可见 **工具调用进度**（如「调用 TFS…」）
- **停止生成**：对话进行中点击「停止」
- **危险操作**（改状态、写文件等）：弹出黄色确认条，点「确认执行」或「取消」

### 6.3 首页卡片

- **今日 TFS 概览**：待处理 / BUG / 超期数量
- **我的研发任务**：列表展示，点「刷新」重新拉取
- 数据经 Hermes 调 TFS MCP；MCP 未配置时显示空状态

### 6.4 定时任务（晨间简报示例）

进入 **日程待办 → 新建定时任务**：

| 字段 | 示例 |
| --- | --- |
| 提示词 | 拉取我今天的超期 TFS 需求并生成简短提醒摘要 |
| 计划(cron) | `0 9 * * 1-5`（工作日 9:00） |

创建后可在列表中 **立即执行 / 暂停 / 恢复 / 删除**。

### 6.5 MCP 工具前置

以下功能依赖 Hermes 侧 MCP 已配置并在线：

| 功能 | 依赖 |
| --- | --- |
| TFS 任务/看板 | TFS MCP |
| 知识库检索 | Obsidian MCP（Vault + Local REST API 27124） |
| 发布流水线（后续） | DevOps MCP |

在 **能力面板** 查看 `/v1/toolsets` 是否列出对应工具。

## 7. 安全须知

1. **不要把 `HERMES_API_KEY` 发给他人或写进前端代码**
2. Hermes 保持 `API_SERVER_HOST=127.0.0.1`，仅本机访问
3. 可选：在 `.env` 设置 `BFF_AUTH_TOKEN`，启用后请求需带 Header `X-Cockpit-Token`
4. 归档笔记时禁止写入明文密码、PAT、Token

## 8. 常见问题

### Q1：顶栏显示「Hermes 离线」

**原因**：`hermes gateway` 未运行，或 Windows 访问不到 WSL2 端口。

**处理**：

1. WSL2 内执行 `hermes gateway`
2. WSL2 内 `curl http://127.0.0.1:5173/health`
3. Windows 再测 `curl http://127.0.0.1:5173/health`
4. 仍失败：考虑在 WSL2 内同时运行 BFF，或检查 WSL 网络/防火墙

### Q2：BFF 启动报 `EADDRINUSE :5180`

**原因**：5180 被旧进程占用。

**处理**（PowerShell）：

```powershell
Get-NetTCPConnection -LocalPort 5180 | ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }
npm run dev
```

### Q3：前端端口不是 5174

Vite 配置了 `strictPort: true`。若 5174 被占用会直接报错，需先释放该端口。

### Q4：对话无响应或一直转圈

1. 能力面板确认 Hermes 在线
2. 确认 Hermes 已配置可用 LLM 提供商
3. BFF 终端查看是否有 `hermes_error` 日志

### Q5：首页/TFS 无数据

1. Hermes 在线但 TFS MCP 未接入 → 在 Hermes 配置 MCP
2. 模型返回非 JSON → 首页可能显示空列表或 raw 回退文本

### Q6：`.env` 改了不生效

重启 `npm run dev`。BFF 在启动时加载环境变量。
